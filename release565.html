Crypto++ 5.6.5

Crypto++ 5.6.5 was released on October 11, 2016. The 5.6.5 release was mostly a maintenance release. The release included two CVE fixes.

The first, CVE-2016-7420, was a procedural finding due to external build systems failing to define NDEBUG for release builds. The procedural gap was a failure to tell users to define it. The second, CVE-2016-7544, was a potential memory corruption on Windows platforms when using Microsoft compilers due to use of _malloca and _freea.

Release Notes

The release notes for Crypto++ 5.6.5 follows.

  • maintenance release
  • expanded community input and support
    • 24 unique contributors for this release
  • fixed CVE-2016-7420 (Issue 277, NDEBUG for production/release)
  • fixed CVE-2016-7544 (Issue 302, _malloca and _freea)
  • Visual Studio project file cleanup
  • improved X86 and X64 MSBuild support
  • added ARM-based build support
  • improved Testing and QA
    • expanded platforms and compilers
    • expanded Coverity into OS X and Windows platforms
    • added Windows test scripts using Strawberry Perl
  • ported to MSVC 2015 SP3, Xcode 7.3, Sun Studio 12.5, GCC 7.0, MacPorts GCC 7.0, Clang 3.8, Intel C++ 17.00

    Minor Bug Fixes and Issues

    The minor bug fix and issue list for Crypto++ 5.6.5 follows. Most issues are tracked for auditing and C&A purposes, but the list may not be complete. A number in parenthesis is the GitHub Issue number, if it was tracked. Sometimes a Git commit is referenced, but many trivial GitHub commits are omitted. Missing Issue numbers or lack of consecutiveness usually indicates feature requests and "won't fix/can't fix" type reports.

    The list below has about 20 issues. The project's test scripts, cryptest.sh and cryptest.nmake, uncovered about 16 (80.0%) of them.

    • Document need for NDEBUG in production/release (CVE-2016-7420, Issue 277)
    • Potential memory corruption when using _malloca and _freea (CVE-2016-7544, Issue 302)
    • Missing CryptoPP::memcpy_s on MinGW32 (Issue 319)
    • Visual Studio 2010 project file cleanup (Issue 314)
    • Improved MSBuild support, including future ARM-based builds (Issue 316)
    • Cygwinn, Newlib and broken compile when -std=XXX due to <signal.h> (Issue 315)
    • Sun Studio 12.2-12.4 and failed debug compile due to <signal.h> (Issue 289)
    • Approximately 80 miscellaneous Coverity findings on Windows (Issues 310-12, all minor or false positives)
    • Approximately 20 miscellaneous Coverity findings on OS X (Issues 295 and 299, all minor or false positives)
    • Solaris GCC and "error: constructor priorities are not supported" (Issue 289)
    • Additional test scripts for Windows testing (Issue 303)
    • SunCC 12.2 - 12.5 and incorrect results for GCM when using SSE2 ASM (Issue 284)
    • SunCC, integer.cpp and "Types cannot be declared in anonymous union" (Issue 274)
    • Clang 3.3, i686 and "error: unknown use of instruction mnemonic without a size suffix (Issue 264)
    • MARS max keysize is limited at 32 bytes (Issue 278)
    • CMake improvements (Issues 276)
